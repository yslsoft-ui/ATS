# Triage Labels

이 프로젝트에서 사용하는 5가지 표준 심사(Triage) 라벨 정의입니다. 에이전트는 이 라벨을 사용하여 작업의 우선순위와 상태를 결정합니다.

| 라벨 명칭 | 의미 |
| :--- | :--- |
| `needs-triage` | 아직 검토되지 않은 새로운 이슈입니다. 관리자의 판단이 필요합니다. |
| `needs-info` | 정보가 부족하여 작업을 진행할 수 없습니다. 사용자(리포터)의 답변을 기다립니다. |
| `ready-for-agent` | 모든 요구사항이 명확히 정의되었습니다. 에이전트가 단독으로 작업을 시작할 수 있습니다. |
| `ready-for-human` | 에이전트가 처리하기에 너무 복잡하거나, 사람의 직접적인 개입이 필요한 작업입니다. |
| `wontfix` | 여러 이유로 인해 현재 프로젝트에서 처리하지 않기로 결정한 작업입니다. |

에이전트가 "상태를 변경하라"는 요청을 받거나 스스로 판단할 때, 위 표의 우측 컬럼에 있는 라벨 문자열을 사용합니다.
이 설정은 `scratch/` 내의 마크다운 이슈 파일 상단 `Status:` 필드에 적용됩니다.
