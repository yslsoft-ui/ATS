# Domain Docs

에이전트가 이 프로젝트의 도메인 지식과 설계 결정을 어떻게 학습하고 활용해야 하는지에 대한 규칙입니다.

## 탐색 전 필수 확인 문서

- **`CONTEXT.md`**: 프로젝트 루트에 위치하며, 프로젝트의 핵심 용어와 비즈니스 로직을 설명합니다.
- **`docs/adr/`**: 과거의 중요한 기술적 결정사항들이 기록되어 있습니다. 새로운 코드를 작성할 때 기존 결정을 위반하지 않는지 확인해야 합니다.

만약 위 파일들이 존재하지 않는다면, 에이전트는 이를 조용히 인지하고 넘어갑니다. 필요할 경우 `/grill-with-docs` 스킬을 통해 문서화를 제안할 수 있습니다.

## 용어 사전 활용

이슈 제목, 제안서, 테스트 이름 등을 작성할 때 반드시 `CONTEXT.md`에 정의된 용어를 사용해야 합니다. 동의어를 임의로 사용하지 마세요.
정의되지 않은 새로운 개념이 필요하다면, 이는 `CONTEXT.md`를 업데이트해야 한다는 신호입니다.

## 설계 결정(ADR) 충돌 감지

작성 중인 코드나 제안이 기존 ADR과 충돌한다면, 이를 묵인하지 말고 명시적으로 보고해야 합니다.

> *주의: ADR-0007(이벤트 소싱)과 충돌할 수 있는 구조입니다. 하지만 현재 성능 최적화를 위해 다음과 같은 이유로 예외 처리를 제안합니다...*

## 파일 구조

이 프로젝트는 **단일 컨텍스트(Single-context)** 구조를 따릅니다.

```
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-chart-engine-change.md
│   └── 0002-websocket-protocol.md
└── src/
```
