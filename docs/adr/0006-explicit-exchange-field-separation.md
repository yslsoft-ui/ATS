# ADR 0006: 거래소 필드 명시적 분리 및 심볼 정규화

## 상태
승인됨 (2026-05-16)

## 맥락
기존 시스템은 `UPB-BTC`, `KIS-005930`과 같이 심볼 문자열에 거래소 접두어를 포함하여 시장을 구분하였습니다. 이 방식은 초기 구현 속도는 빠르지만, 다음과 같은 장기적 한계가 있습니다:
1. 프론트엔드에서 표시를 위해 매번 접두어를 제거하는 정규식 처리가 필요함.
2. 서로 다른 거래소에서 동일한 심볼(예: 'CORE')을 가질 경우 문자열 파싱 로직이 복잡해짐.
3. 데이터베이스 정규화 원칙에 어긋나며, 시장별 쿼리 성능이 저하될 수 있음.

## 결정
모든 데이터 구조와 통신 규격에서 **거래소(Exchange)**와 **종목(Symbol)** 필드를 명시적으로 분리합니다.

1. **데이터 스키마 변경**:
   - `candles`, `orders_history` 테이블에 `exchange` 컬럼 추가.
   - `candles` 테이블의 기본키를 `(exchange, symbol, interval, timestamp)`로 변경.
2. **심볼 정규화**:
   - 모든 `symbol` 필드에는 거래소에서 제공하는 순수 코드(예: `BTC`, `005930`)만 저장.
   - 업비트의 경우 `KRW-BTC` 대신 `BTC`로 저장하고 `exchange` 필드에 'upbit' 저장.
3. **통신 규격**:
   - API 요청 파라미터 및 웹소켓 메시지에 `exchange` 필드 필수 포함.

## 결과
- **장점**: UI 로직 단순화, DB 조회 성능 향상, 다중 거래소 확장 시 무결성 보장.
- **단점**: 기존 코드 및 데이터베이스에 대한 대대적인 리팩토링 및 마이그레이션 필요.
