# ADR 0004: 다중 시장 대응을 위한 통합 인증 관리 체계 도입

## Status
Proposed (2026-05-16)

## Context
기존 시스템은 업비트(가상자산) 전용 API 키만 관리하면 되었으나, 국내 주식(한국투자증권 KIS API)이 추가됨에 따라 다음과 같은 인증 복잡도가 발생하였습니다.
- 한국투자증권은 매번 Access Token을 발급받아야 하며 유효기간이 24시간으로 짧음.
- 웹소켓 연결을 위한 별도의 Approval Key 관리가 필요함.
- 실전 투자와 모의 투자 서버의 엔드포인트 및 인증 정보가 상이함.

이러한 인증 로직이 각 수집기(Collector)나 실행기(Executor)에 분산될 경우 코드 중복과 보안 취약점이 발생할 우려가 있습니다.

## Decision
시스템 전역에서 사용할 수 있는 **`CredentialProvider`** 싱글톤 모듈을 도입합니다.

1.  **중앙 관리**: 모든 거래소의 API 키, 보안 토큰, 승인 키를 한곳에서 관리합니다.
2.  **자동 갱신**: 유효기간이 있는 토큰(KIS Access Token 등)의 경우, 만료 전 자동으로 재발급을 수행하여 컴포넌트들에게 항상 유효한 인증 정보를 공급합니다.
3.  **추상화**: 수집기나 실행기는 구체적인 인증 방식을 몰라도 `credential_provider.get_token('kis')`와 같은 단순한 인터페이스로 인증 정보를 획득합니다.

## Consequences
- **Positive**: 인증 로직의 집중화로 보안 관리가 용이해지고 컴포넌트 간 결합도가 낮아집니다.
- **Negative**: 인증 서버 장애 시 전역 시스템이 영향을 받을 수 있으므로 적절한 캐싱 및 예외 처리가 필요합니다.
