# ADR 0011: 엄격한 가격 Hydration 및 튜플 키 기반 조회 강제화 (Strict Price Hydration and Tuple Key Lookup)

## 상태
**승인됨 (Approved)**

## 컨텍스트
기존 시스템은 전략 서비스 기동 시 필요한 모든 종목에 대한 실시간 시세가 즉시 확보되지 않는 경우, 성과 평가 및 리스크 한도 검사 등 하위 계산 계층에서 임의의 폴백(Fallback)(예: 로컬 DB 재조회, 외부 거래소 API 조회, 혹은 포지션의 평균 매수 단가 `avg_price` 사용)을 수행하여 비정상 상황을 우회하려 시도했습니다. 
이로 인해 다음과 같은 문제점들이 노출되었습니다.
1. **정합성 훼손**: 실거래 환경에서 시세 수집기 장애 등으로 최신 시세가 없을 때, 오래된 DB 데이터나 평단가로 잘못 계산된 평가금에 근거해 오동작이 유발될 위험이 큽니다.
2. **복합 키 비표준화**: 메모리 캐시 및 현재가 사전의 키가 튜플 키 `(exchange_id.lower(), symbol)`와 문자열 키 `"exchange:symbol"` 또는 단순히 `"symbol"` 형태로 혼용되어 코드 상 가독성이 떨어지고 버그의 원인이 되었습니다.
3. **디버깅 난이도**: 하위 모듈에서 조용히 기본값으로 덮어씌우는(Silent Fallback) 로직으로 인해 초기 구동 시점의 데이터 누락 사실을 빠르게 인지하기 어려웠습니다.

## 결정
1. **초기 Hydration 일원화 및 Fail-Stop 도입**:
   - 전략 서비스(`StrategyService`) 기동(`start()`) 시점에 실질 보유 포지션(`quantity > 0`) 및 활성 전략 엔진 종목의 최신 종가 캔들을 데이터베이스에서 일괄 배치 조회(`get_latest_closed_candles_batch`)하여 메모리 캐시(`latest_prices`)에 Hydrate합니다.
   - DB에 가격이 아예 없거나, DB 최신 캔들의 타임스탬프와 현재 시각의 오차가 `stale_threshold` (초 단위 또는 밀리초 단위 매칭 적용)를 초과하는 경우, 임의 폴백을 금지하고 즉시 `KeyError`를 발생시켜 데몬 기동을 즉시 중단(Fail-Stop)합니다.
2. **복합 튜플 키 표준화**:
   - 가격 조회를 위한 사전의 키는 무조건 `(exchange_id.lower(), symbol)` 복합 튜플 키만 사용하도록 표준화합니다.
3. **계산 레이어 내 폴백 전면 제거**:
   - `PortfolioManager`, `PerformanceAnalyzer`, `ExecutionScorer` 등의 계산 레이어 내에서 DB/API 쿼리 및 평균 매수단가(`avg_price`)로의 임의 폴백 로직을 전면 삭제하고 오직 메모리 캐시의 복합 튜플 키 strict 조회만을 유지합니다.
4. **청산 완료 종목 가격 노출 왜곡 방지**:
   - 청산 완료 종목(`quantity == 0`)의 평가용 가격(`final_price`)은 `0.0`으로 고정하여 시스템의 자산 가치를 올바르게 평가하게 하되, 화면에 현재가가 0원으로 오인 노출되지 않도록 `display_price` 및 `price_source` 필드를 통해 참고용 가격을 격리하여 반환합니다.

## 결과 및 영향
- 최신 시세 데이터가 보장되지 않는 비정상적인 구동 환경을 원천 차단하여 실거래 안전성이 극대화됩니다.
- 계산 레이어가 메모리 캐시 조회로 고도로 단순화되고 Stateless 설계 원칙을 준수하게 되어 성능 및 예측 가능성이 향상됩니다.
- 시세 누락이나 stale 상태가 발생할 경우 즉시 기동 크래시가 발생하므로, 운영팀에서 시세 수집기나 DB 동기화 오류를 Fail-Fast로 인지할 수 있습니다.
