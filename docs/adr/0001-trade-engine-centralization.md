# 0001. TradeEngine 도입을 통한 로직 중앙집권화

* Status: accepted
* Date: 2026-05-15

## Context and Problem Statement

기존 시스템에서는 실시간 데이터 수집기(`CollectorManager`)와 과거 데이터 검증기(`BacktestEngine`)가 각각 별도의 캔들 생성 및 전략 실행 로직을 가지고 있었습니다. 이로 인해 다음과 같은 문제가 발생했습니다:
1. 실시간 매매와 백테스트 결과 사이의 로직 불일치 가능성.
2. 동일한 전략 로직을 두 곳에서 중복 관리해야 하는 유지보수 부담.
3. 새로운 지표나 전략 추가 시 테스트 복잡도 증가.

## Decision Outcome

모든 매매 관련 핵심 로직(캔들 생성, 기술 지표 계산, 전략 평가, 신호 발생)을 `TradeEngine`이라는 단일 클래스로 캡슐화하기로 결정했습니다.

### 세부 내용
1. **TradeEngine**: 종목별로 인스턴스화되며, `process_tick` 메서드를 통해 모든 입력을 처리합니다.
2. **StrategyRegistry**: 전략 객체 생성을 표준화하여 엔진 주입을 용이하게 합니다.
3. **Warm-up Process**: 엔진 초기화 시 과거 데이터를 주입하여 지표 상태를 동기화하는 과정을 표준화했습니다.

## Consequences

* **Positive**: 실시간과 백테스트 로직이 100% 동일함을 보장할 수 있습니다. 전략 개발 및 테스트 주기가 단축됩니다.
* **Negative**: 각 종목별로 엔진 인스턴스를 관리해야 하므로 메모리 사용량이 다소 증가할 수 있습니다.
* **Neutral**: 시스템 구조가 데이터 수집-처리-실행의 명확한 계층으로 분리되었습니다.
