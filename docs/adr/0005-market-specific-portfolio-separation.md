# ADR 0005: 시장별 독립 포트폴리오 운영 및 통합 뷰 구현

## Status
Proposed (2026-05-16)

## Context
국내 주식 시장은 가상자산 시장과 비교했을 때 다음과 같은 차이점이 있습니다.
- **결제 주기**: 주식은 T+2 결제 원칙이 적용되나, 코인은 즉시 결제됨.
- **세금 및 수수료**: 증권거래세 등 주식 특유의 비용 구조가 존재함.
- **운영 시간**: 주식은 정규장 외 시간 외 거래 등이 복잡함.

이 모든 자산을 하나의 포트폴리오 객체에서 관리할 경우, 자산 평가 및 수익률 계산 로직이 극도로 복잡해지고 데이터 오염의 위험이 있습니다.

## Decision
시장 유형별로 독립된 **`Portfolio`** 인스턴스를 운영하고 관리합니다.

1.  **독립 운영**: `StockPortfolio`와 `CryptoPortfolio`를 별도의 인스턴스(혹은 DB 레코드)로 관리하여 각 시장의 특수한 회계 규칙을 충실히 따릅니다.
2.  **포트폴리오 매니저 확장**: `PortfolioManager`는 여러 개의 포트폴리오를 관리하며, 실행 신호의 접두어(예: `KIS-`)를 보고 적절한 포트폴리오를 선택합니다.
3.  **통합 뷰(Consolidated View)**: UI 계층에서는 각 포트폴리오의 합계를 계산하여 사용자에게 전체 자산 현황을 제공합니다.

## Consequences
- **Positive**: 각 시장의 도메인 로직을 명확히 분리하여 정확한 수익률 계산이 가능해집니다.
- **Negative**: 여러 계좌에 걸친 복합 주문(예: 비트코인을 팔아 삼성전자를 매수)과 같은 시나리오 처리가 다소 복잡해질 수 있습니다.
