# PRD: 평가 데몬 모니터링 텔레메트리 복구 및 DB 정합성 확보

## 1. 개요
- **배경**: 데몬 모니터링 메뉴 내 "평가 데몬" 탭에서 평가 데이터가 전혀 표시되지 않고 Stale 상태로 남아 있는 현상이 발생함.
- **목적**: 백그라운드 리스너 및 실거래 주문 동기화 도중 발생하는 무한 루프 반복 에러를 제거하고, 꼬여 있던 DB 외래키 스키마 정정 및 누락된 과거 평가 데이터를 백필하여 모니터링 데이터 렌더링을 완전히 복구함.

## 2. 주요 기능 및 요구사항
- **ZMQ 텔레메트리 복구**: FastAPI 웹서버에서 ZMQ 리스너 내부의 UnboundLocalError를 유발하는 잘못된 로컬 모듈 임포트 구조 정정.
- **실거래 KIS 주문 동기화 복구**: KIS 주문 동기화 중 `datetime.datetime` 인스턴스에 존재하지 않는 `timedelta` 속성 호출 오류 해결.
- **DB 스키마 정정 및 데이터 복구**: 
  - `proposal_evaluations` 테이블의 외래키 참조를 올바르게 `strategy_proposals(id)`로 정정.
  - 기존에 생성되었던 `strategy_proposals` 데이터(101~111번)에 매칭되는 `proposal_evaluations` 레코드들을 settings.yaml 에 정의된 Horizons 규격으로 일괄 자동 생성(Backfill)함.
- **실시간 평가 처리 및 모니터링**: 백필 즉시 백그라운드 `shadow_eval_daemon`에 의해 평가 연산이 100% 실행되도록 보장하여, 모니터링 메뉴로 데이터가 원활히 연동되도록 함.
